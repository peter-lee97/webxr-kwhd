# Fix Wand Visibility in Desktop View

## Issue
The wand does not appear when toggled in desktop view. For desktop view, the wand should appear in a first-person view visually.

## Current Implementation Analysis
- Wand is created as a Three.js mesh with rod and feather components
- Wand is attached to the camera in desktop mode
- Wand visibility is controlled by the toggle function
- Wand is positioned at (0, 0, -2) relative to the camera
- The wand may be outside the camera's field of view or not properly oriented

## Root Cause
The wand is positioned directly in front of the camera along the Z-axis, which may not be visible due to:
1. Camera's field of view and perspective projection
2. Incorrect positioning relative to the camera's view direction
3. Wand orientation not matching the expected first-person view

- Adjust wand position to be visible in first-person view

- Ensure wand doesn't obstruct the main view
- Account for camera's perspective projection

### 2. Improve Wand Visibility Toggle

- Add visual feedback when wand is active
